v1.14.0 · Multi-Agent 编排

Multi-Agent
编排系统

8 个专业 Agent · 4 种协作模式 · 覆盖全部 36 个 Skill
像组建顶级工程团队一样，让 AI Agent 协同完成复杂工程任务

── 4 种协作模式

⬇️ sequential
顺序执行
各 Agent 按角色顺序依次执行，每步完成后输出传递给下一个 Agent。经典分工协作，适合有明确执行链的工程任务。
Architect → Coder → Reviewer → DevOps
适用场景：dev-workflow · full-dev-cycle · new-feature
⚡ parallel
并行分析
各 Agent 独立并行输出自己负责维度的分析，最后由"汇总委员会"整合为完整报告。多视角审查，速度最快。
Architect ⟶ Reviewer ⟶ Security ⟶ → 汇总整合 ✓
适用场景：架构评审 · security-audit · 多维度分析
🔄 review-loop
审查迭代
Coder 实现 → Reviewer 审查（通过/修改意见）→ Coder 修改，最多 2 轮迭代。实现与质量的闭环协作。
Coder 实现 → Reviewer 审查 → Coder 修改 → ✅ 最终版
适用场景：quality-workflow · bugfix-workflow
🏛️ consensus
共识决策
多个 Agent 各自独立给出方案，最后 Consensus 阶段取交集、推荐最优解。适合有争议的架构决策与技术选型。
Architect 方案A Coder 方案B → 🏛️ Consensus → 📋 最优解
适用场景：solution-design · tech-research · 架构决策

那些问题

我已经在用 Cursor / Claude Code 了，Ethan 是不是多余的？
不冲突，恰恰相反——Ethan 让你的 AI 工具变得更好用。

Cursor / Claude Code 是"对话工具"，告诉 AI 要做什么；Ethan 是"标准化协议"，让 AI 知道怎么做才专业。安装后，你在 Cursor 里输入「代码审查」，AI 会自动按照 OWASP Top 10、Blocker/Major/Minor 分级来输出，而不是靠随机发挥。
我们团队能自定义 Skill 吗？可以加团队自己的规范吗？
完全支持。在项目根目录创建 .ethan/skills/ 目录，放入 .yaml 或 .md 文件即可定义自定义 Skill。

比如你们有专属的 Code Review Checklist、部署规范、或者安全审查流程——都可以写成 Skill，让全团队通过统一触发词使用。ethan skill new my-skill 可以快速生成模板。
怎么让全团队都用上？需要每个人单独配置吗？
推荐把 ethan install 生成的配置文件提交到 git 仓库。这样新同学 clone 项目后，配置自动就有了，不需要每个人单独跑命令。

Ethan 本身需要全局安装一次（npm install -g ethan-skill），可以加到团队 Onboarding 文档里。
支持哪些 AI 工具？我用的工具在列表里吗？
目前支持 11 个平台：Cursor（新 .mdc 格式）、Cursor（老 .cursorrules）、VS Code Copilot、Cline、Claude Code、Windsurf、Zed、JetBrains AI、Continue、通义灵码、腾讯 CodeBuddy。

运行 ethan install 时会自动检测你的项目环境并选择对应格式；也可以用 ethan install --platform cursor 手动指定。
免费吗？有没有收费项目？
核心功能完全免费，开源在 npm。36 个 Skills、10 个 Pipelines、31 个 MCP 工具、Multi-Agent 编排——全部免费使用。

未来计划推出团队版（Skills 云同步、DORA 指标面板、合规工具），但现在的个人和团队使用没有任何费用。
